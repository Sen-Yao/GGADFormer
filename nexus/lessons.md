# Nexus Lessons Learned

## 2026-03-19

### 1. Reddit 数据集 Seed 敏感性极高
- 同样超参数下，不同 seed 的 AUC 差异高达 5% (0.5351 vs 0.5874)
- 论文报告的 0.5782 可能是多次实验的最佳结果
- **教训**: 对于小样本/不平衡数据集，必须进行多 seed 验证

### 2. Elliptic 超参数调整失败
- 原问题: batch_size=1024 导致梯度不稳定
- 尝试修复: batch_size=4096, lr=0.0001
- 结果: 更差 (AUC 0.28 vs 原 0.7058)
- **可能原因**: 学习率过低导致模型陷入局部最优
- **教训**: 超参数调整需要系统化实验，不能仅凭直觉

### 3. 小 Batch Size 的问题
- Elliptic: train_rate=0.1, 总样本 4656, batch_size=1024 → 每 epoch 仅 4.5 批次
- 导致梯度估计不稳定
- **教训**: 计算 batch 数量，确保每个 epoch 有足够多的梯度更新

### 4. 重构损失 (rec_loss) 监控
- Elliptic 崩溃时 rec_loss 占总损失 99.9%
- 正常情况下应该监控各损失分量的比例
- **教训**: 设置 rec_loss 阈值监控，防止模型崩溃

### 5. 多 GPU 并行实验效率
- 成功在 5 个 GPU 上并行运行 5 个 seed
- 每个实验约 50 秒，总时间约 2 分钟
- **教训**: 利用多 GPU 进行超参数扫描和多 seed 验证非常高效
